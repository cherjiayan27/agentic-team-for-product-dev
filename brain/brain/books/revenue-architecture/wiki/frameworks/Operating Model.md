# Operating Model

## Definition

Layer 4 of the [[Six-Layer Model Stack]]. The first of the three dynamic layers. Combines the static [[Bowtie]] system with a standardised interface ([[SPICED]]) that lets all [[Means and Methods|Means and Methods]] — BANT, MEDDIC, ICP, SPIN, ABM, NPS, SEO, etc. — plug in coherently. Prevents the exponential-interconnection cost explosion that destroys GTM efficiency at $20–150M ARR.

## The OS analogy

Like a software operating system, the Operating Model has three components (Figure 8.4 / 8.5):

| Software OS | GTM Operating Model |
|---|---|
| **System** (kernel, services) | The [[Bowtie]] / [[Data Model]] |
| **Interface** (APIs) | [[SPICED]] |
| **Apps** | [[Means and Methods]] (the tools and methodologies teams use) |

## The problem it solves

Without an Operating Model (Figure 8.1):
- Departments operate in silos
- Each hire imports methods from their previous company
- Each vendor introduces its own vocabulary
- Each executive (18-month tenure in fast-growing SaaS) leaves a methodology layer
- By $150M ARR: 12+ GTM motions, exponential interconnections, declining productivity

Adding a new GTM motion **squares** (not doubles) the number of interconnections (Figures 8.2 → 8.3). Costs escalate; productivity drops; growth stalls.

## The fix: standardised interface

[[SPICED]] as a common language across all GTM motions, functions, and roles:

- **S**ituation — the existing state
- **P**ain — the dysfunction
- **I**mpact — the desired outcome (the anchor)
- **C**ritical Event — the deadline
- **D**ecision — how the choice is made (via the [[Buying Center]])

## Market leaders use Operating Models externally

- **AWS** — cloud infrastructure OS
- **Salesforce AppExchange** — CRM OS with app ecosystem
- **Adobe Creative Cloud** — design/creation OS

Revenue Architecture adapts the same pattern **internally** — the GTM org as an OS.

## Role in the stack

- Built on [[Mathematical Model]] — uses Growth Formula metrics
- Built on [[Data Model]] — uses the Bowtie as its canonical system
- Feeds [[Growth Model]] and [[GTM Model]] — both depend on the Operating Model's interface for cross-motion coherence

## Rules

- **Operating Model required when operating ≥2 GTM motions.** Introducing a second motion without one triggers the interconnection explosion.
- **Adoption requires enforcement.** Once standardised, changes go through a change-request process (per [[Data Model]] governance rules).
- **Standardise the interface, not every tool.** Teams can still pick the right Means/Method for their motion — they just all plug into the same SPICED interface.

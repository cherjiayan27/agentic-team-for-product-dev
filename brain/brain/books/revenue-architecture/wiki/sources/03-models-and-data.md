# Chapter 03 — Models and Data

**Source:** `raw/markdown/03-models-and-data.md` (book pages 70–87)

## Thesis

Models are simplified representations of reality. No single model is 100% accurate; **combining models** increases accuracy. Civil engineers use 6 models to build a suspension bridge; human anatomy needs skeletal + muscular + nervous + cardiovascular models to describe how a body works. A recurring revenue business similarly needs **6 essential models**, each addressing a distinct dimension, structured as a hierarchy — the [[Six-Layer Model Stack]].

## Key claims

- **Don't blindly follow GTM trends.** Every trend comes with tools, content, and unfamiliar disciplines. Before implementing, model the trade-offs: *What worked? What didn't? Will it work for us?*
- **Models come from science.** Scott Page's *The Model Thinker* is the intellectual reference. Models have revolutionised civil engineering, medicine, and economics for centuries; GTM is the laggard.
- **Bridges are built from 6 component models** (piers, anchors, towers, main cable, suspension cables, deck — Figure 3.1). Engineers scale each mathematically, combine them, stress-test. This is how the Golden Gate Bridge was designed in 1935 and still works in 2025.
- **Human anatomy needs multiple overlapping models** (skeletal, muscular, nervous, cardiovascular — Figure 3.2). No single model describes human movement; combined they do.
- **The same applies to recurring revenue.** Six models:
  1. [[Revenue Model]] — monetization strategy
  2. [[Data Model]] — Bowtie + Data Structure
  3. [[Mathematical Model]] — nonlinearity + Growth Formula
  4. [[Operating Model]] — SPICED + Means and Methods
  5. [[Growth Model]] — S-curve + Revenue Breakpoints
  6. [[GTM Model]] — 5 Touch GTM Motions
- **Structural hierarchy** (Figure 3.4):
  - Models 1–3 (Revenue, Data, Mathematical) = **Static**. Set once; changes are rare and high-impact.
  - Models 4–6 (Operating, Growth, GTM) = **Dynamic**. Change frequently in response to conditions.
- **Data organisation across the hierarchy** (Figure 3.5). Reading top-down:
  - **Market penetration** ← GTM metrics ← GTM Model ← Growth and Costs
  - **Growth metrics** ← Growth Model ← Growth Rate
  - **Productivity metrics** ← Operating Model ← Growth Formulas
  - **Conversion metrics (VM/CR/Δt)** ← Mathematical Model ← Data Model
  - **Pricing and payment structure** ← Data Model ← Revenue Model
- **The 3 Es** for informed decisions:
  - **Effectiveness** (Scalable) — is the goal being achieved?
  - **Efficiency** (Sustainable) — minimal wasted resources
  - **Economy** (Durable) — optimized quality outcomes
- Data-driven decision-making = first principles + data + math. All three required to move beyond gut-feel.

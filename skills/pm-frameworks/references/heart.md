# HEART Framework

**When relevant:** Choosing success metrics for UX-oriented features. Measuring product quality from the user's perspective.

---

## The 5 Dimensions

| Metric | Definition | Example Signals |
|--------|------------|-----------------|
| **Happiness** | Subjective satisfaction | NPS, CSAT, survey scores, perceived quality |
| **Engagement** | Depth of involvement | Session length, actions/session, interaction frequency |
| **Adoption** | New user behavior | Signups, feature activation, first-time use |
| **Retention** | Continued usage | D7/D30 retention, churn rate, return behavior |
| **Task Success** | Efficiency and effectiveness | Completion rate, time-on-task, error rate |

## Goals-Signals-Metrics Process

1. **Goal**: What UX outcome indicates success?
2. **Signal**: How would success manifest in observable behavior?
3. **Metric**: How do we measure the signal?

## Example

```
Feature: New checkout flow

Goal: Users complete purchases faster
Signal: Reduced time in checkout, fewer drop-offs
Metrics:
  - Median checkout time (target: <2 min)
  - Checkout completion rate (target: 85%)
  - Error rate (target: <2%)
```

## When to Use HEART

| Situation | Recommended |
|---|---|
| UX and product quality optimization | **HEART** |
| Company-wide strategic alignment | North Star |
| Early growth and funnel bottlenecks | AARRR |
